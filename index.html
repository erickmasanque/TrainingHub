Technical Support Process Flow

Azure SQL Database for Copilot Studio (Developer Support)

Phase 1

Initial Setup & Configuration

  • Verify SQL DB Setup: Check service tier and collation settings.
  • Network & Firewall: Confirm client IP is allowed and Azure services can access.
  • App Registration: Validate service principal exists and has DB user mapping.
  • Connection String: Assist with format and advise on using Key Vault for secrets.
Phase 2

Data Ingestion (One-Time)

  • Storage Account Access: Ensure data file exists and permissions (Storage Blob Data Reader) are set.
  • Review Ingestion Method: Check tool used (e.g., Azure Data Factory, Function App).
  • Troubleshoot Function App: Review triggers, dependencies, and application settings.
  • Verify Data Schema: Query target table post-ingestion for correctness and header mapping.
Phase 3

Copilot Studio Integration

  • Check Copilot Connector: Verify SQL Server connector and its authentication setup.
  • Troubleshoot Read Access: Test the exact query from Copilot directly against the DB.
  • Support Write-Back: Advise on using Stored Procedures and grant necessary permissions.
  • Debug API Calls: Review Power Automate flow history for detailed error messages.
Phase 4

General Troubleshooting

  • Analyze Error Logs: Use Azure Monitor and Application Insights for detailed logs.
  • Isolate Problem: Systematically test each component in the chain to find the failure point.
  • Consult Documentation: Refer to official Microsoft Learn docs for best practices.
  • Escalate if Unresolved: Pass to a senior dev with a clear summary of steps taken.

Resolution & Documentation

Document the solution in the knowledge base to help with future requests.
